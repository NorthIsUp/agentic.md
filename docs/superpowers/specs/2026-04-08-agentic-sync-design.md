# agentic-sync Design Spec

A Rust CLI that syncs Claude Code project configuration (rules, skills, MCP servers) to other AI coding tool formats. Claude is the source of truth; other tools receive generated files.

## Targets

Currently supported (more planned):

| Target | Flag value |
|---|---|
| Cursor | `cursor` |
| GitHub Copilot | `copilot` |

## CLI Interface

```
agentic-sync [--check] [--fix] [--pr] [--out=<targets>] [--overwrite] [path]
```

### Flags

| Flag | Behavior |
|---|---|
| `--check` | Compare generated output to disk. Exit 0 if in sync, 1 if stale, 2 if parse error. Lists stale files to stderr. Default mode if no flag given. |
| `--fix` | Write/overwrite target files. Skips existing files without `generated-by: agentic-sync` header (warns to stderr). |
| `--overwrite` | Used with `--fix`. Overwrites target files regardless of `generated-by` header. |
| `--pr` | Like `--check` but outputs a markdown diff/summary to stdout suitable for posting as a PR review comment (e.g. via `gh pr comment`). |
| `--out=<targets>` | Comma-separated list of targets to generate. Can be repeated: `--out=cursor --out=copilot`. Default: all supported targets. |
| `path` | Project root. Defaults to cwd. |

### Exit Codes

| Code | Meaning |
|---|---|
| 0 | In sync (check) or success (fix) |
| 1 | Stale files detected (check) or write failures (fix) |
| 2 | Parse error in source files |

## Source Discovery

The tool looks for these files at the project root:

| Source | Purpose |
|---|---|
| `CLAUDE.md` | Project rules and instructions |
| `.claude/rules/*.md` | Additional rule files |
| `.claude/skills/*/SKILL.md` | Skills with frontmatter |
| `.mcp.json` | MCP server configuration |

No walking up directories. No reading `~/.claude/` (global config). Project-level only.

## Output Structure

Given sources, the tool generates:

```
.cursor/rules/_project.mdc          # CLAUDE.md preamble (before first ##)
.cursor/rules/{heading-slug}.mdc    # One per ## section in CLAUDE.md
.cursor/rules/{name}.mdc            # One per .claude/rules/*.md
.cursor/skills/{name}.mdc           # One per .claude/skills/*/SKILL.md
.cursor/mcp.json                    # Transformed from .mcp.json
.github/copilot-instructions.md     # Full CLAUDE.md content verbatim
```

## CLAUDE.md Parsing

### Section Splitting

CLAUDE.md is split on `## ` headings. Each heading becomes a separate Cursor rule file.

- **Slug generation:** `## Commit Conventions` becomes `commit-conventions.mdc`
- **Preamble:** Content before the first `## ` heading goes to `.cursor/rules/_project.mdc`
- **Empty sections:** Skipped, no file generated.
- **Duplicate slugs:** `## Testing` appearing twice produces `testing.mdc` and `testing-2.mdc` with a warning.

### Per-Section Frontmatter Overrides

Sections can include a fenced frontmatter block with tool-namespaced overrides:

```markdown
## Testing
---
cursor: alwaysApply=false
cursor: globs=**/*.test.ts
---
Run tests with...
```

Each `cursor:` line is stripped of the `cursor: ` prefix and mapped to a field in the Cursor `.mdc` frontmatter. Unknown namespaces are ignored (reserved for future targets).

The frontmatter block is consumed by the tool and **not** included in the generated rule content.

### Default Cursor Rule Output

```yaml
# Generated by agentic-sync — do not edit manually
---
generated-by: agentic-sync
alwaysApply: true
description: "Commit Conventions"
---

Use conventional commits...
```

- `generated-by: agentic-sync` is always added to track ownership.
- `alwaysApply: true` is the default, overridable via `cursor:` frontmatter.
- `description` defaults to the heading text, overridable via `cursor:` frontmatter.

### Copilot Output

The full CLAUDE.md content is copied verbatim into `.github/copilot-instructions.md` with a generated-by HTML comment at the top:

```markdown
<!-- Generated by agentic-sync — do not edit manually -->

(full CLAUDE.md content here)
```

## .claude/rules/*.md

Each file in `.claude/rules/` becomes an additional `.cursor/rules/{name}.mdc`. If the file has YAML frontmatter with `cursor:` keys, they are mapped the same way as CLAUDE.md sections. The `description` field from Claude's frontmatter carries over directly.

## Skills Mapping

Each `.claude/skills/*/SKILL.md` becomes `.cursor/skills/{name}.mdc`.

### Frontmatter Mapping

| Claude SKILL.md field | Cursor .mdc field |
|---|---|
| `name` | filename |
| `description` | `description` |
| `paths` | `globs` (if supported, otherwise dropped) |
| `disable-model-invocation: true` | `alwaysApply: false` |
| (default) | `alwaysApply: true` |
| `cursor: *` | stripped of prefix, passed through |

All other Claude-specific fields (`allowed-tools`, `context`, `agent`, `model`, `effort`, `hooks`, `shell`) are dropped — no Cursor equivalent.

Markdown body copies verbatim.

## MCP Config

`.mcp.json` at project root is transformed to `.cursor/mcp.json`.

The schemas are nearly identical. The tool normalizes to Cursor's expected shape (wrapping in `mcpServers` if needed). Environment variable interpolation (`${VAR}`) passes through as-is — both tools use the same syntax.

## Ownership & Conflict Handling

### Generated-by Header

Every generated file includes `generated-by: agentic-sync` in its YAML frontmatter (or as an HTML comment for non-YAML files like copilot-instructions.md).

### Write Protection

- **`--fix`**: If a target file exists and **lacks** the `generated-by: agentic-sync` marker, it is **skipped** with a warning to stderr.
- **`--fix --overwrite`**: Writes regardless of the marker.
- **`--check`**: Files without the marker that conflict with a would-be generated file are reported as warnings, not failures.

### Cleanup

On `--fix`, any file in `.cursor/rules/` or `.cursor/skills/` that **has** the `generated-by: agentic-sync` marker but does not correspond to a current source is **deleted**.

Hand-written files without the marker are never deleted.

## Logging

### Standard Mode

Plain text to stderr: warnings for skipped files, errors for parse failures, info for what was written.

### GitHub Actions Mode

Detected via `GITHUB_ACTIONS=true` environment variable. Outputs GitHub annotations:

- Errors: `::error file={path}::{message}`
- Warnings: `::warning file={path}::{message}`
- Groups: `::group::`/`::endgroup::` for verbose output

## hk Integration

```pkl
["agentic-sync"] {
    glob = List("CLAUDE.md", ".claude/**", ".mcp.json")
    check = "agentic-sync --check"
    fix = "agentic-sync --fix"
}
```

This runs the tool as a pre-commit linter via hk. `--check` for validation, `--fix` for auto-repair.

## Future Targets

The architecture supports adding targets by:

1. Adding a new `--out` value
2. Implementing a generator that reads the parsed intermediate representation and writes target-specific files
3. Supporting `{target}: key=value` frontmatter namespace

Planned targets: Codex (`AGENTS.md`), Gemini (`GEMINI.md`).

## Non-Goals

- Global/user-level config sync (`~/.claude/settings.json`)
- Bidirectional sync (detecting Cursor edits and syncing back)
- Watch mode / daemon
- Config file for the tool itself (convention over configuration)
